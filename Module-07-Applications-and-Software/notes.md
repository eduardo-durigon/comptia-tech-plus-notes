# Module 7: Applications and Software

Overview of application software, productivity tools, software distribution, licensing models, installation methods, and troubleshooting processes.

---

## 1. What Is Software

Software is a **set of instructions that tells a computer what to do**.

The **operating system (OS)** reads and executes these instructions.

Software is typically written for a **specific operating system** such as Windows, macOS, Linux, Android, or iOS.

Two main categories of software:

### System Software

Includes software that manages hardware and basic system operations.

Examples:

- Operating systems
- Device drivers
- Utility programs

Drivers act as **translators between hardware and the operating system**.

### Application Software

Programs designed to help users perform tasks.

Examples:

- Web browsers
- Email clients
- Office software
- Messaging applications
- Media editing tools

---

# 2. Web Browsing Software

Web browsers allow users to **access and navigate websites on the internet**.

Common browsers:

- Google Chrome
- Mozilla Firefox
- Safari
- Microsoft Edge

Key points:

- Browsers work across different operating systems.
- Users can **sign into an account** to sync bookmarks, passwords, and settings across devices.
- Mobile browsers often have **simplified interfaces** due to smaller screens.

---

# 3. Email Software

Email systems send messages **from one server to another** across the internet.

Two main types:

### Client-Based Email

Email is accessed through an installed application.

Example:

- Microsoft Outlook

The software connects to the email server to **send and retrieve messages**.

### Web-Based Email

Email is accessed through a **web browser**.

Examples:

- Gmail
- Yahoo Mail
- Outlook Web

Many modern services operate using **Software as a Service (SaaS)**, where emails are stored on the provider’s servers.

---

# 4. Messaging Software

# 10. Safely Managing Software

Software used to be distributed through **physical media such as floppy disks, CDs, and DVDs**.

Today it is usually downloaded from the internet.

Best practice:

Always download software from **trusted sources**.

Trusted sources include:

- Official developer websites
- Official app stores

Examples of app stores:

- Apple App Store
- Google Play Store
- Microsoft Store
- Amazon Appstore

Apps can often be **re-downloaded later using the same account**.

---

# 11. Software Licensing

Most software requires users to accept a **license agreement**.

This is known as the:

**EULA (End User License Agreement)**

Common licensing models:

### Single User License

Software installed on **one computer**.

### Site License

Organizations can install software **on multiple computers**.

### Subscription Model

Users pay a **monthly or yearly fee**.

Benefits:

- Continuous updates
- Security patches
- Access to new features

---

# 12. Open Source vs Proprietary Software

## Open Source Software

Source code is publicly available and can be modified.

Examples:

- Linux
- LibreOffice

Advantages:

- Free to use
- Community development
- Customizable

---

## Proprietary Software

Owned by a company and protected by copyright.

Examples:

- Microsoft Office
- Adobe software
- Oracle products

Users must **purchase a license and follow usage terms**.

---

# 13. Software Installation

Software installation methods depend on the operating system.

## Windows

Software is commonly installed using:

`.exe` files (executables)

Installed programs are typically stored in:

- Program Files
- Program Files (x86)

The **x86 folder contains older 32-bit applications**, while modern systems run 64-bit programs.

---

## macOS

Software may be distributed as:

- `.dmg` disk image files
- App Store downloads

Installation usually involves **dragging the application into the Applications folder**.

---

## Linux

Most software is installed through **package managers and repositories**.

Software packages are downloaded and installed from trusted repositories maintained by the Linux distribution.

---

# 14. Removing Software

Unused or outdated software should be removed to improve:

- Security
- Performance
- Storage space

Reasons to uninstall software:

- Security vulnerabilities
- No longer receiving updates
- Unused applications
- Background processes consuming RAM

Examples of removal methods:

Windows:

- Settings → Apps → Uninstall
- Third-party tools such as Revo Uninstaller

macOS:

- Drag the application to the Trash

---

# 15. Bloatware

Bloatware refers to **unnecessary software bundled with other programs**.

Problems with bloatware:

- Uses storage space
- Consumes system resources
- May introduce security risks

It is recommended to **remove unnecessary bundled applications**.

---

# 16. The CompTIA Troubleshooting Process

The CompTIA troubleshooting methodology follows **six steps**:

1. Identify the problem  
2. Establish a theory of probable cause  
3. Test the theory  
4. Establish a plan of action and implement the fix  
5. Verify full system functionality  
6. Document findings

During troubleshooting it is important to:

- Interview the user
- Check recent system changes
- Review logs and documentation
- Test possible solutions
- Escalate issues when necessary

# Key Takeaways

- Software consists of instructions that tell computers how to perform tasks.
- Application software helps users complete specific activities such as browsing, messaging, and productivity work.
- Productivity tools include word processors, spreadsheets, and presentation software.
